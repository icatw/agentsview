# Paraglide I18n Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Replace `svelte-i18n` with Paraglide JS while preserving current locale selection, persistence, and translated UI behavior.

**Architecture:** Paraglide compiles flat underscore source message JSON files from `messages` into generated message/runtime modules under `src/lib/paraglide`. The generated output stays ignored and is regenerated by the Vite plugin or the `i18n:compile` script before standalone type checks. The app uses Paraglide's `["localStorage", "preferredLanguage", "baseLocale"]` strategy with the existing `agentsview-locale` key, while a small handwritten i18n facade preserves Svelte reactivity and existing helper tests.

**Tech Stack:** Svelte 5, Vite+, Vitest, Paraglide JS, TypeScript.

  const runtime = await import("../paraglide/runtime.js");

  setLocale("zh-CN");

  expect(runtime.getLocale()).toBe("zh-CN");
  expect(localStorage.getItem(LOCALE_STORAGE_KEY)).toBe("zh-CN");

- [x] **Step 2: Replace facade implementation**

Keep `DEFAULT_LOCALE`, `LOCALE_STORAGE_KEY`, `SUPPORTED_LOCALES`,
`normalizeLocale()`, and `chooseInitialLocale()`. Replace `svelte-i18n` imports
with Paraglide runtime imports and implement:

```ts
export function setLocale(value: SupportedLocale) {
  setParaglideLocale(value);
  try {
    localStorage?.setItem(LOCALE_STORAGE_KEY, value);
  } catch {
    // Ignore storage failures; the active in-memory locale still changes.
  }
}

export function initI18n() {
  setLocale(chooseInitialLocale());
}
```
